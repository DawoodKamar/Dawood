## Spacing Scale

Declared values (multiples of 4 — extracted from existing Tailwind classes in source):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, badge padding |
| sm | 8px | p-2 / gap-2 | Button padding, compact gaps |
| md | 16px | p-4 / gap-4 | Default container padding (px-4) |
| lg | 24px | p-6 / gap-6 | Card padding, section internal gaps |
| xl | 32px | py-8 | Main content top/bottom padding |
| 2xl | 48px | py-12 | Homepage section spacing |
| 3xl | 64px | py-16 / py-24 | Hero section padding |

Exceptions:
- Touch target minimum: 44px height on all interactive elements (RESP-03). Achieve via `min-h-[44px] flex items-center` on nav links and footer links. This is a Phase 4 addition — existing nav buttons already have `p-2` (8px padding) on icon buttons; text links need audit and padding uplift.
- Max content width: `max-w-4xl` (896px) on all page wrappers — not a spacing token but a layout constraint. Do not widen.
- Max prose width: `max-w-2xl` on positioning/body text sections — kept narrower than container for readability.

## Typography

Extracted from existing source (BaseLayout.astro, index.astro, [id].astro, Footer.astro):

| Role | Size | Tailwind | Weight | Tailwind | Line Height | Usage |
|------|------|----------|--------|----------|-------------|-------|
| Body | 16px | text-base (default) | 400 regular | font-normal | 1.5 (Tailwind prose default) | Body copy, card descriptions, footer copyright |
| Label | 14px | text-sm | 400 regular | font-normal | 1.5 | Post metadata (date, reading time), footer nav, category badges |
| Heading | 24–32px | text-2xl / text-3xl | 700 bold | font-bold | 1.2 | Section headings (h2), post titles on listing page |
| Display | 36–60px | text-4xl / text-5xl / text-6xl | 700 bold | font-bold | 1.1 | Hero h1 (responsive: 36px mobile → 48px md → 60px lg) |

**Constraint (RESP-02):** No `text-xs` (12px) on body content. `text-xs` is permitted only on category badge labels where context is sufficient. All body copy must be `text-base` (16px) minimum at every breakpoint.

**Prose typography:** Blog post body content rendered via `@tailwindcss/typography` (`prose dark:prose-invert max-w-none`). Do not override prose scale — it handles heading hierarchy inside markdown.

**Weight contract:** Exactly 2 declared weights:
- 400 regular (`font-normal`) — body copy, labels, descriptions
- 700 bold (`font-bold`) — headings, display, CTA button labels, active nav links use `font-semibold` (600) as the only permitted exception

## Interaction Contracts

### Tap Target Requirement (RESP-03)
All tappable elements must have a minimum hit area of 44×44px.

| Element | Current State | Required Fix |
|---------|---------------|--------------|
| Desktop nav links | `py-1` implicit height | Add `min-h-[44px] flex items-center` or `py-3` |
| Mobile nav links | `block py-1` | Change to `block py-3` (24px top+bottom = 48px total) |
| Theme toggle button | `rounded-lg p-2` on 20×20 icon = ~36px | Change `p-2` to `p-3` (12px padding = 44px total: 20+24) |
| Hamburger button | `rounded-lg p-2` on 20×20 icon = ~36px | Change `p-2` to `p-3` |
| Footer nav links | `text-sm` no padding | Add `py-3 inline-flex items-center` |
| Social links in footer | `text-sm` no padding | Add `py-3 inline-flex items-center` |
| Newsletter submit button | Not audited — add `min-h-[44px]` | Confirm in Plan 04-02 |
| Contact form submit button | Not audited | Confirm `min-h-[44px]` in Plan 04-02 |
| Prev/next post navigation | Not audited | Confirm `min-h-[44px]` in Plan 04-02 |

### Image Handling Contract (SEO-07, SEO-08, RESP-04)
- All `<img>` tags replaced with `<Image />` from `astro:assets`. Raw `<img>` produces no srcset and causes CLS.
- RESEARCH.md confirmed: no raw `<img>` tags currently exist in `src/` — hero is text-only. Audit is precautionary.
- Hero/above-fold images: `loading="eager" fetchpriority="high"` attributes required.
- All other images: default `loading="lazy"` (Astro Image default).
- Remote images in markdown: specify explicit `width` and `height` to prevent CLS.

### Dark Mode Contract
- Toggle via `.dark` class on `<html>` element. Already implemented via `astro:before-swap` listener.
- No changes in Phase 4. Confirmed: dark mode carries through view transitions without FOUC.

### View Transition Contract
- `<ClientRouter />` from `astro:transitions` already active. No changes in Phase 4.
- BaseHead.astro renders inside `<head>` — does not interfere with `ClientRouter`.
